This document provides specific examples and pointers for each of the top issues.


🔍 ISSUE #1: MISSING ARRAY/OBJECT METHODS (2,985 failures)
───────────────────────────────────────────────────────────────────────────────

ERROR PATTERN: CallProperty: callee is not a function (got Undefined)

WHAT THIS MEANS:
  Code is trying to call a method that doesn't exist or returned undefined.
  Example: Array.from() is undefined when it should be a function

WHERE IT HAPPENS:
  • Most Array static methods: Array.from(), Array.isArray(), Array.of()
  • Most Object static methods: Object.keys(), Object.values(), Object.entries()
  • Many Object.prototype methods
  • Many Array.prototype methods

HOW TO FIND SPECIFIC MISSING METHODS:
  1. Run test and look at failure message
  2. Message shows which method is undefined
  3. Search src/builtins/ for where it should be implemented

EXAMPLES TO START WITH:
  • Array.isArray - Very basic, commonly used
  • Array.from - Frequently tested
  • Object.keys - Extremely common
  • Object.values - Common
  • Object.entries - Common

CHECK THESE FILES:
  • src/builtins/array.rs - Array methods implementation
  • src/builtins/object.rs - Object methods implementation
  • src/builtins/array_prototype.rs - Array.prototype methods
  • src/builtins/object_prototype.rs - Object.prototype methods


🔍 ISSUE #2: EXCEPTION HANDLING (2,411 failures)
───────────────────────────────────────────────────────────────────────────────

ERROR PATTERN A: Expected a throw [but code executed without throwing]
COUNT: 1,114 failures

WHAT THIS MEANS:
  Code ran successfully when it should have thrown an error.
  Example: delete of private property should throw SyntaxError but doesn't

COMMON CASES:
  • Using invalid syntax that should parse error
  • Operating on undefined in strict mode (should throw)
  • Deleting non-configurable properties (should throw)
  • Missing validation checks

CHECK THESE FILES:
  • src/parser/mod.rs - Parser validation
  • src/compiler/mod.rs - Compiler error checks
  • src/runtime/error.rs - Error throwing


ERROR PATTERN B: expected parse SyntaxError, got: This statement should not be evaluated
COUNT: 1,297 failures

WHAT THIS MEANS:
  Parser accepted invalid syntax and executed it.
  Should have failed at parse time, not runtime.

COMMON CASES:
  • Invalid assignment targets
  • Unsupported syntax not caught during parsing
  • Missing syntax validation

CHECK THESE FILES:
  • src/parser/expression.rs - Expression parsing
  • src/parser/statement.rs - Statement parsing
  • src/parser/assignment.rs - Assignment target validation


ERROR PATTERN C: unsupported assignment target
COUNT: 681 failures combined (404 as parse error + 277 as internal error)

WHAT THIS MEANS:
  Can't assign to certain targets (like function calls).
  Example: (foo()) = bar should not be allowed

CHECK THESE FILES:
  • src/parser/assignment.rs
  • Look for assignment target validation


🔍 ISSUE #3: OBJECT.GETOWNPROPERTYDESCRIPTOR (1,764 failures)
───────────────────────────────────────────────────────────────────────────────

ERROR PATTERN: The overriden version of Object.getOwnPropertyDescriptor was called!

WHAT THIS MEANS:
  When code overrides Object.getOwnPropertyDescriptor via a Proxy or property,
  the override is being called but something is wrong with the result.

POSSIBLE ROOT CAUSES:
  1. Proxy/override mechanism is broken
  2. Descriptor object being returned is malformed
  3. Property flags not being set correctly
  4. The override check is triggering incorrectly

WHERE IT HAPPENS:
  • Any code that uses Object.getOwnPropertyDescriptor
  • Code that defines custom versions via Proxy
  • Code testing descriptor behavior

DEBUGGING STRATEGY:
  1. Find a simple failing test for getOwnPropertyDescriptor
  2. Check what descriptor object is returned
  3. Verify all properties (value, writable, enumerable, configurable)
  4. Check if override is being called at all/always

CHECK THESE FILES:
  • src/builtins/object.rs - Object.getOwnPropertyDescriptor implementation
  • src/interpreter/property_descriptor.rs - PropertyDescriptor struct
  • src/vm/proxy.rs - Proxy handler logic


🔍 ISSUE #4: SYMBOL.TOSTRING (112 failures)
───────────────────────────────────────────────────────────────────────────────

ERROR PATTERN: Cannot convert a Symbol value to a string

WHAT THIS MEANS:
  Code tried to convert a Symbol to string, which should work.
  Example: String(Symbol('test')) should work in some contexts

WHERE IT HAPPENS:
  • String concatenation with symbols
  • String constructor with symbols
  • Template literals with symbols

FIX:
  Implement Symbol.toString() or Symbol string conversion

CHECK THESE FILES:
  • src/builtins/symbol.rs - Symbol implementation
  • src/vm/value.rs - Value.toString() implementation


🔍 ISSUE #5: OBJECT.KEYS OVERRIDE (89 failures)
───────────────────────────────────────────────────────────────────────────────

ERROR PATTERN: The overriden version of Object.keys was called!

SIMILAR TO: Issue #3 (Object.getOwnPropertyDescriptor)

This is the same root cause:
  • Proxy/override mechanism for Object.keys is broken
  • The override is being called but returning wrong result

CHECK SAME FILES:
  • src/builtins/object.rs - Object.keys implementation
  • src/vm/proxy.rs - Proxy handler logic

                        DEBUGGING WORKFLOW
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Pick one issue to focus on
  Recommended: Start with Issue #3 (Object.getOwnPropertyDescriptor)
  Why: Single largest error count (1,764), likely a focused fix

STEP 2: Find a small failing test
  • Search test262_verbose6.log for the error
  • Find a test file that exhibits the issue
  • Example test might be:
    test262/test/built-ins/Object/getOwnPropertyDescriptor/basic.js

STEP 3: Run just that test
  ./target/release/stator_test262 --test262-dir test262 \
    --filter "getOwnPropertyDescriptor"

STEP 4: Review the test code
  • Open the test file in text editor
  • Understand what it's testing
  • Run it with debug output if possible

STEP 5: Trace through implementation
  • Find where Object.getOwnPropertyDescriptor is implemented
  • Add debug logging to see what's happening
  • Check what descriptor object is being returned
  • Verify proxy/override mechanism

STEP 6: Write fix
  • Make minimal change to fix the issue
  • Re-run the test to verify
  • Run full test suite to ensure no regressions

STEP 7: Repeat for next high-impact issue

COMMON PATTERNS FOR EACH ISSUE TYPE:

Missing Method Implementation:
  1. Find builtin class (e.g., Object, Array)
  2. Add new method to prototype/static methods
  3. Implement logic that returns correct value

Override/Proxy Issue:
  1. Check src/vm/proxy.rs for proxy handler logic
  2. Verify proxy trap is being called
  3. Check return value is correct

Parser Validation Issue:
  1. Find validation location in src/parser/
  2. Add check that throws error for invalid syntax
  3. Ensure error message is correct
